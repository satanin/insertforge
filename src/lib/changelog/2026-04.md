### v1.1.1

#### Counter tray fit fixes

- fixed constrained counter trays so reducing token counts is allowed even when the tray is already wider than its fixed available space
- added regression coverage for constrained counter tray edits inside layered boxes

### v1.1.0

#### Project metadata and exports

- added project-level `Name` in `Dimensions`
- project JSON, STL ZIP, 3MF and PDF exports now use the project name for filenames
- formalized InsertForge project exports with `schemaVersion: 3` and `appVersion`
- legacy Counter Slayer `version: 2` projects are migrated to the InsertForge schema on import
- refreshed InsertForge favicon assets

#### Game container planning

- added `Height` to the `Game Container` dimensions
- empty height means `Auto`, calculated from the sum of all layer heights
- project/layer summary now displays width, depth and effective height
- `Preview sizes` in project view now shows a global Game box height indicator when height is manually set
- the indicator reports used height and remaining/over height while preserving the exploded layer view
- layer height calculations now use the visible lid height so boxes and trays align with the assembled height shown in the planner
- adding or moving the first tray into a placeholder `Empty box` now resets placeholder dimensions back to auto-sizing

#### Counter tray fit fixes

- edge-loaded counter tray stack clearance is now applied to the full stack instead of once per token gap
- counter tray bottom emboss avoids extraction cutouts so names are not clipped by finger scoops
- regression coverage added for counter tray stack clearance and project metadata behavior

#### Text and print workflow fixes

- STL export now includes separate lid text inlay geometry when boxes use `Text mode: Inlay`
- plain ASCII text now preserves the original JSCAD vector text path unless accent handling is needed
- export cleanup was adjusted to preserve original geometry behavior for existing text workflows
- regression coverage added for lid text modes, box visible height and placeholder empty-box conversion

### v1.0.0

First independent InsertForge release after the fork from Counter Slayer. This version groups the work that turned the fork into a distinct product instead of continuing the original app's numbering.

#### Product identity

- visible rebrand to `InsertForge`
- attribution kept to Counter Slayer and Dave Snider
- own in-app versioning shown in the browser UI
- own changelog surfaced from the app header

#### Layered Box

- new `Layered Box` object type
- internal layers with independent sections
- support for `counter`, `cardDraw`, `cardDivider`, `cardWell`, `cup` and `playerBoard` sections
- internal manual layout editing per layer
- moving sections between internal layers
- visual alignment of shell, lid, explode and editor flow with regular `Box`
- support for fused solid internal layers, filler and relief cutouts
- support for print options and `lid text inlay`
- support for exporting layered-box contents correctly as independent printable parts

#### New tray and content support

- `Miniature Rack` tray type with dedicated editor and preview improvements
- `Player Board` support and filtered shape handling
- `Empty box` hollow mode
- support for labels, accented text and emboss/inlay improvements

#### Layout and planning improvements

- `Edit layout` support recovered and expanded for layer workflows
- `Board` support in layer layout editing
- `Layered Box` support in normal layer layout editing
- gap previews and size overlays in layer view
- object duplication actions
- navigation between layers without unnecessary geometry regeneration

#### Fit and spacing behavior

- `Adapt to gap` for `Box`
- `Adapt to gap` for `Layered Box`
- improved rules when contents can shrink vs only expand
- fixes for preserving fixed sizes when adding sections
- fixes for tray fit handling inside fixed layered boxes

#### Export and print reliability

- `3MF` export extended to include layered-box sections and assemblies
- invalid `3MF` object names are now sanitized before export
- topology validation now checks exported geometry for naked or non-manifold edges
- planar holes can now be repaired automatically during export cleanup when possible
- fixes for layered-box solid-fill export

#### Quality and regression control

- fixes for incorrect tray selection/edit application
- fixes for layered-box layout selection and section visibility
- first `Vitest` regression suite added for:
  - layered-box selection and internal layout behavior
  - export name sanitization
  - planar hole repair for export geometry
  - visible app version staying aligned with `package.json`
