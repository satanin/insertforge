### v1.1.39

#### Render performance and view transitions

- geometry generation now uses scoped render requests for the selected piece, selected layer or full project instead of regenerating the whole project for every view
- selected trays, selected box assemblies, layer views and the full project view now keep separate geometry caches so navigation can reuse the right cached result
- returning from a layer view to the project view now restores the full-project cache instead of reusing partial layer geometry
- switching views now keeps the previous scene visible and shows `Preparing view` feedback while cached or newly generated geometry is applied
- added lightweight development timing logs for geometry worker and apply phases

#### Miniature Rack editor

- `Miniature Rack` now uses the same split editor flow as other complex trays, with separate `Rack` and `Slots` tabs
- the rack settings and slot list can render independently, making the editor easier to scan for racks with many slots

### v1.1.37

#### Project reset

- added a `Reset project` action in `Dimensions` for starting over from a blank InsertForge project
- reset clears generated geometry and project data together so stale previews are not reused after the project is replaced
- added regression coverage for resetting project state and restoring default global settings
