> Upstream Counter Slayer history before the InsertForge fork. InsertForge's own release line starts at `v1.0.0`.

#### Optional honeycomb infill

Added an optional honeycomb infill pattern for box lids and bottoms. When enabled, this pattern reduces filament usage by approximately 40-60% while maintaining structural integrity. [#45](https://github.com/Siege-Perilous/counterslayer/pull/45)

#### Layers

Added the concept of layers — horizontal, same height slices — to Counter Slayer. Multiple boxes and loose trays can exist within the same layer and can be arranged within a layout. This layout has no impact on your STL geometry but is helpful when trying to visualize how your boxes and trays sit vertically within your game box. The contents of each layer will normalize tray and box heights to produce a flat top.

#### 3MF export

Added 3MF export option alongside STL export. 3MF packages all models (boxes, lids, and trays) into a single file with objects pre-arranged on the build plate so they don't overlap. [#32](https://github.com/Siege-Perilous/counterslayer/pull/32)

#### Manual box layouts

Drag and drop trays within a box to create custom arrangements. Click "Edit Layout" in the box view to enter edit mode, then drag trays to position them. Trays snap to edges and each other for precise alignment. [#29](https://github.com/Siege-Perilous/counterslayer/pull/29)

#### Cup trays

Added a new **cup tray** type for storing loose game components like dice, tokens, and meeples. Cup trays use a flexible split-based layout system—start with a single cup and split it vertically or horizontally to create custom arrangements with mixed cup sizes. Drag dividers to resize cups, with snap-to-align for precise positioning. [#28](https://github.com/Siege-Perilous/counterslayer/pull/28) [#34](https://github.com/Siege-Perilous/counterslayer/pull/34)

#### Card trays

Added two new tray types for playing cards. **Card draw trays** hold a single stack of cards on a sloped floor for easy retrieval with a finger cutout. **Card divider trays** store multiple stacks of cards standing on edge, separated by walls, with per-stack floor heights for mixed card sizes. Both support standard card sizes (Standard, Mini American, Mini European, Euro, Japanese, Tarot, and Square). Custom card sizes can be defined and are shared globally across all trays. [#21](https://github.com/Siege-Perilous/counterslayer/pull/21)

#### Tray rotation for efficient bin packing

Trays can now be automatically rotated 90° to achieve more efficient packing within the box. The algorithm considers both orientations and picks the one that minimizes total box footprint. You can override the automatic rotation per-tray if needed, forcing a specific orientation. [#15](https://github.com/Siege-Perilous/counterslayer/pull/15)
